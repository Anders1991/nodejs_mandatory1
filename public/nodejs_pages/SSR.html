SSR

What is it?

SSR is short for "Serverside Rendering" as opposed to CSR which is short for "Clientside Rendering". Serverside Rendering causes the server to load the page before it sends it to the user (as opposed to CSR where the page is loaded in the browser on the clients machine).

Below is a list of pros and cons of SSR

Pros

Faster initial access

The pages are rendered immediately, even on slow internet connections because the browser doesn't have to load the javascript before it sends the page.

Better SEO

Search engines don't have to run your javascript before they index your site.

Cons

Pricier servers

SSR requires better and bigger servers than CSR, because it is more dependant on internet speed.

High user traffic = slower for everyone
